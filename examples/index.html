1. Import and Initialization

Add these components into your pages.
Remember to change the config settings if changes are made to the id attribute on either of the components.

1.1. Import

1.1.1 Import (Standard HTML)

Include the cookie-manager.js tag on your page. It will need to be served as part of your project as there is no centralised source/CDN for this script.

HTML

                    <script src="./cookie-manager.js"></script>

1.1.2 Import (Node Module)

Require the node-module within your client-side JavaScript.

HTML

                    const cookieManager = require('@hmcts/cookie-manager');

1.2. Initialization

Add the configuration settings into the head of each page.
Include the logic to enable/disable consent within a function to be called on user-preferences being saved

JavaScript

                /*  This is where you'd perform logic regarding granting or denying the injection
                *   of cookies, scripts and tags that required consent. This callback is called
                *   with an object containing the current cookie preferences.
                */
                const updateConsent = function (cookieStatus) {
                    // Google Tag Manager Analytics Consent
                    const gtag = window.gtag || function () { dataLayer.push(arguments); };
                    if (cookieStatus.analytics === 'on') {
                        gtag('consent', 'update', {'analytics_storage': 'granted'});
                    } else {
                        gtag('consent', 'update', {'analytics_storage': 'denied'});
                    }

                    // Dynatrace RUM Consent
                    const dtrum = window.dtrum;
                    if (cookieStatus.apm === 'on') {
                        dtrum.enable();
                        dtrum.enableSessionReplay();
                    } else {
                        dtrum.disableSessionReplay();
                        dtrum.disable();
                    }
                }


                /*  In your Cookie Manager config, it should be set to link
                *   to the callbacks written for the save callback (if any)
                */
                const cookieManagerConfig = {
                  'user-preference-cookie-name': 'cm-user-preferences',
                  'user-preference-saved-callback': updateConsent,          // Function that is called when the user preferences for cookies are saved.
                  'cookie-manifest': [
                    {
                      'category-name': 'essential',
                      'optional': false,
                      'cookies': [
                        'cm-user-preferences'
                      ]
                    },
                    {
                      'category-name': 'analytics',                         // Add for Google Analytics
                      'optional': true,
                      'cookies': [
                        '_ga',
                        '_gid',
                        '_gat_UA-XXXXXXX-X'                                 // Update this to service's GA id.
                      ]
                    },
                    {
                      'category-name': 'apm',                               // Add for Dynatrace RUM / APM
                      'optional': true,
                      'cookies': [
                        'dtCookie',
                        'dtLatC',
                        'dtPC',
                        'dtSa',
                        'rxVisitor',
                        'rxvt'
                      ]
                    }

                  ]
                };

                /*  Initializes the cookie manager with the provided settings */
                cookieManager.init(cookieManagerConfig);

Add the following callbacks to your code and update your Cookie Manager config to setup the multi-stage aspect of the banner.

JavaScript

                        /*  This callback is called when the 'accept' action is fired within the cookie banner
                        *   We are hiding the first stage (decision) and showing the second stage (confirmation of consent) in this function.
                        */
                        const bannerAcceptCallback = function () {
                            document.querySelector('.cm-cookie-banner__decision').hidden = true;
                            document.querySelector('.cm-cookie-banner__confirmation').hidden = false;
                        }


                        /*  This callback is called when the 'reject' action is fired within the cookie banner
                        *   We are hiding the first stage (decision) and showing the second stage (confirmation of no-consent) in this function.
                        */
                        const bannerRejectCallback = function () {
                            document.querySelector('.cm-cookie-banner__decision').hidden = true;
                            document.querySelector('.cm-cookie-banner__confirmation').hidden = false;
                        }


                        /*  This callback is called when the banner save action is fired within the cookie banner (caused by accepting or rejecting).
                        *   This is where you'd likely call a method to that would programmatically remove or add the
                        *   consent-required scripts/tags back onto the page.
                        */
                        const bannerSaveCallback = function (cookieStatus) {
                            updateConsent(cookieStatus); // updateConsent function should be used to enable/disable analytics scripts, tags and injections.
                        }


                        /*  In your Cookie Manager config, it should be set to link to the callbacks written for the
                        *   accept/reject callback, save callback and to disable auto-hide. If auto-hide is left default (enabled), the
                        *   banner will automatically hide after the first stage, not providing any confirmation message.
                        */
                        const cookieManagerConfig = {
                            ...
                            "cookie-banner-id": "cm-cookie-banner",                           // Set to the same id as the cookie-banner wrapper div
                            "cookie-banner-visible-on-page-with-preference-form": false,      // Disables Cookie Banner on preference page.
                            "cookie-banner-auto-hide": false,                                 // Value 'true' will disable second-stage.
                            "cookie-banner-accept-callback": bannerAcceptCallback,
                            "cookie-banner-reject-callback": bannerRejectCallback,
                            "cookie-banner-save-callback": bannerSaveCallback,
                            ...
                        };
