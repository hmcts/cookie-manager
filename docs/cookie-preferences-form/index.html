Cookie preferences form

HTML / Nunjucks template

Here is an example of a cookie preferences form (utilising the GOV.UK Frontend ‘Radios’ component). Code has been provided for both HTML and Nunjucks on how this can be used to create a cookie preferneces form for use with @hmcts/cookie-manager. See the GOV.UK Frontend Design System for more about this component

How the cookie preferences form is handled by @hmcts/cookie-manager

The default config for the cookie banner is shown below:

{
  preferencesForm: {
    class: 'cookie-preferences-form'
  }
}

When the cookie preferences form can be found within the DOM, it will be targeted using the class option from the config.

The preferences form needs to be configured so that each radio button has the name attribute set to that of its relevant category, with the value attribute set to on or off to consent or reject the cookies of that category respectively.

On the form submission event, the selected radio options will be parsed and then made the user’s active cookie preferences.

More info about the configuration options can be found in the configuration-options/cookie-preferences-form section.
